# 09_プロジェクト構成.md

# 婚活AIトレーナー

Version: 1.0

---

# 1. 設計方針

本プロジェクトは、AIロジック・画面・データ管理を明確に分離する。

各ディレクトリは単一責任を持ち、依存関係を最小限に抑える。

---

# 2. 採用技術

| 項目       | 技術                    |
| -------- | --------------------- |
| Frontend | Next.js(App Router)   |
| Language | TypeScript            |
| UI       | React                 |
| Style    | Tailwind CSS          |
| DB       | SQLite                |
| ORM      | Prisma                |
| AI       | OpenAI API            |
| 音声認識     | OpenAI Speech-to-Text |
| 状態管理     | React Context（MVP）    |

# 4. ディレクトリ責務

## app/

画面およびAPIルートを管理する。

責務

* 画面表示
* 画面遷移
* API Route

---

## ai/

AI関連処理を集約する。

責務

* OpenAI呼び出し
* Prompt生成
* Structured Output解析
* AIロジック

他ディレクトリからOpenAI APIを直接呼び出してはならない。

---

## prompts/

Markdown形式のプロンプトを保存する。

コードへ直接記述しない。

---

## components/

再利用可能なUI部品。

---

## contexts/

React Context。

MVPではSession情報を保持する。

---

## hooks/

Custom Hook。

例

* useConversation
* useSpeech
* useEvaluation

---

## lib/

共通ライブラリ。

例

* Prisma
* Logger
* Utility

---

## repositories/

DBアクセスのみ担当する。

SQLやPrisma呼び出しを直接画面から行わない。

---

## services/

業務ロジック。

RepositoryとAIを組み合わせる。

例

ConversationService

EvaluationService

---

## types/

TypeScript型定義。

---

## constants/

Enum

固定値

メッセージ

---

## utils/

純粋関数。

副作用を持たない。

---

# 13. 設計思想

AI・画面・データベース・業務ロジックを明確に分離する。

変更されやすいもの（プロンプト・設定・AIロジック）はコードから分離し、保守性と拡張性を重視した構成とする。

本プロジェクトは「AIを利用するアプリ」ではなく、「AI会話エンジンを中核としたアプリケーション」として設計する。
