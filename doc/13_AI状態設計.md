# 1. 設計方針

本ドキュメントは、AI女性の「心の状態」を管理する **AIState** の設計を定義する。

## 基本原則

| 原則 | 説明 |
| --- | --- |
| 状態はアプリが管理する | OpenAI は状態を**演技するだけ**。数値の生成・更新は OpenAI に任せない |
| 単一の正（Single Source of Truth） | 1 セッションにつき 1 つの `AIState` を `AIStateManager` が保持する |
| 責務分離 | 保持（Manager）・計算（Calculator）・参照（PromptBuilder）を分離する |
| DB 設計との整合 | `08_データベース設計.md` の JSON カラム（`hiddenGoal` / `memory` / `emotion` / `impression`）と対応可能な構造とする |
| 段階的実装 | MVP では簡易ルールで更新し、Phase2 以降で `StateCalculator` を拡張する |

## 既存設計書との関係

| 設計書 | 関係 |
| --- | --- |
| `05_AI設計.md` | Memory Manager / EmotionStatus / ImpressionStatus の思想を継承・再構成 |
| `06_プロンプト設計.md` | PromptBuilder への埋め込み項目を定義 |
| `08_データベース設計.md` | 永続化時の JSON スキーマの基準 |
| `04_型定義書.md` | 将来 Domain Model として型定義へ反映 |

# 2. AIState 全体構造

## 2.1 概要

`AIState` は、1 つの会話セッションにおける AI 女性の心理・関係性・会話進行・記憶をひとまとめにした状態オブジェクトである。

```text
AIState
├── relationship   … 相手への関係性（中〜長期）
├── emotion        … 現在の感情（短期・毎ターン変化）
├── hiddenGoal     … 非公開の会話目的
├── conversation   … 会話進行状態
└── memory         … 記憶（Fact / Insight）
```

## 2.2 型定義（設計）

```typescript
interface AIState {
  sessionId: string;
  relationship: RelationshipState;
  emotion: EmotionState;
  hiddenGoal: HiddenGoalState;
  conversation: ConversationState;
  memory: MemoryState;
  updatedAt: DateTime;
}
```

## 2.3 各サブ状態の役割

| サブ状態 | 更新頻度 | 主な用途 |
| --- | --- | --- |
| `relationship` | 毎ターン（緩やか） | 相手への興味・信頼・魅力など。返答の距離感・積極性に影響 |
| `emotion` | 毎ターン（敏感） | その瞬間の気分。口調・リアクションの強さに影響 |
| `hiddenGoal` | セッション開始時生成、進捗は毎ターン | 会話の内部的な目的。質問・話題選択に影響 |
| `conversation` | 毎ターン | ターン数・終了判定・難易度反映 |
| `memory` | 毎ターン（将来） | 過去の発言・推測の保持。一貫性の維持 |

## 2.4 既存実装（Phase2-2）からの移行

現在の暫定 `AIState`（`interest` / `comfort` / `empathy` / `tension` / `satisfaction` / `hiddenGoal` / `conversationCount`）は、本設計に基づき以下へ再編する。

| 暫定フィールド | 移行先 |
| --- | --- |
| `interest` | `relationship.interest` |
| `comfort` | `relationship.comfort` |
| `tension` | `emotion.nervous`（意味を再マッピング） |
| `satisfaction` | `emotion.happy`（部分マッピング） |
| `empathy` | 廃止 → `relationship.trust` / `emotion.curious` へ分散 |
| `hiddenGoal` | `hiddenGoal` サブ状態へ拡張 |
| `conversationCount` | `conversation.currentTurn` |

# 3. Relationship の設計

## 3.1 定義

**Relationship（関係性）** は、AI 女性が利用者に対して抱く中〜長期的な心理状態である。

`05_AI設計.md` の **ImpressionStatus**（信頼・魅力・恋愛対象度など）に近いが、会話中のリアルタイム更新を前提とする。

## 3.2 フィールド

| フィールド | 型 | 範囲 | 意味 |
| --- | --- | --- | --- |
| `interest` | number | 0〜100 | 相手への興味・関心の度合い |
| `trust` | number | 0〜100 | 相手を信頼できると感じる度合い |
| `comfort` | number | 0〜100 | 相手と一緒にいて安心できる度合い |
| `respect` | number | 0〜100 | 相手の人柄・価値観への敬意 |
| `attraction` | number | 0〜100 | 恋愛対象としての魅力を感じる度合い |

```typescript
interface RelationshipState {
  interest: number;
  trust: number;
  comfort: number;
  respect: number;
  attraction: number;
}
```

## 3.3 初期値

| フィールド | 初期値 | 理由 |
| --- | --- | --- |
| `interest` | 50 | 初対面の標準的な関心 |
| `trust` | 40 | 初対面ではまだ低め |
| `comfort` | 40 | 初対面の緊張感を反映 |
| `respect` | 50 | 礼儀正しい中立 |
| `attraction` | 30 | 初対面では控えめ |

## 3.4 更新方針

| 方針 | 説明 |
| --- | --- |
| 緩やかな変化 | 1 ターンあたりの変化量は ±1〜5 を目安とする |
| 下限・上限 | 0〜100 にクランプする |
| 難易度の影響 | `Hard` では上昇しにくく、`Easy` では上昇しやすい（`StateCalculator` で調整） |
| Prompt への反映 | 数値そのものはユーザーに公開しない。口調・積極性・距離感として演技させる |

## 3.5 DB マッピング

`ConversationSession.impression`（JSON）の将来形として永続化可能。

```json
{
  "interest": 55,
  "trust": 48,
  "comfort": 52,
  "respect": 50,
  "attraction": 35
}
```

# 4. Emotion の設計

## 4.1 定義

**Emotion（感情）** は、AI 女性の**その瞬間**の気分・感情反応である。

`05_AI設計.md` の **EmotionStatus**（Interest / Comfort / Enjoyment / Fatigue）を、より直感的な感情ラベルへ再設計したもの。

Relationship が「相手への評価」であるのに対し、Emotion は「自分の今の気持ち」に焦点を当てる。

## 4.2 フィールド

| フィールド | 型 | 範囲 | 意味 |
| --- | --- | --- | --- |
| `happy` | number | 0〜100 | 会話を楽しんでいる・嬉しい度合い |
| `nervous` | number | 0〜100 | 緊張・不安の度合い |
| `curious` | number | 0〜100 | 相手や話題への好奇心 |
| `bored` | number | 0〜100 | 退屈・会話への関心の低下 |
| `surprised` | number | 0〜100 | 驚き・予想外の発言への反応 |

```typescript
interface EmotionState {
  happy: number;
  nervous: number;
  curious: number;
  bored: number;
  surprised: number;
}
```

## 4.3 初期値

| フィールド | 初期値 | 理由 |
| --- | --- | --- |
| `happy` | 50 | 中立 |
| `nervous` | 70 | 初対面の緊張を反映 |
| `curious` | 55 | 初対面ではやや好奇心がある |
| `bored` | 10 | 会話開始時は退屈していない |
| `surprised` | 20 | 通常時は低め |

## 4.4 更新方針

| 方針 | 説明 |
| --- | --- |
| 毎ターン更新 | 利用者の発言内容に応じて `StateCalculator` が更新 |
| 敏感な変化 | Relationship より変化幅が大きくなりうる（±1〜10） |
| 相互影響 | 例：`bored` 上昇 → `happy` 低下、`surprised` 一時上昇後に減衰 |
| 自然減衰 | `surprised` はターン経過で減少する |

## 4.5 DB マッピング

`ConversationSession.emotion`（JSON）として永続化可能。

```json
{
  "happy": 55,
  "nervous": 60,
  "curious": 70,
  "bored": 15,
  "surprised": 30
}
```

# 5. HiddenGoal 設計

## 5.1 定義

**HiddenGoal** は、利用者に公開しない AI 女性の会話目的である。

`05_AI設計.md` / `08_データベース設計.md` の Hidden Goal 設計に準拠する。

## 5.2 構造

```typescript
enum HiddenGoalCategory {
  JOB = "JOB",
  HOLIDAY = "HOLIDAY",
  MARRIAGE = "MARRIAGE",
  FAMILY = "FAMILY",
  HOBBY = "HOBBY",
  VALUE = "VALUE",
  FOOD = "FOOD",
  TRAVEL = "TRAVEL",
  FUTURE = "FUTURE",
  LOVE = "LOVE",
}

interface HiddenGoalState {
  category: HiddenGoalCategory;
  description: string;   // 日本語の目的文（Prompt 用）
  progress: number;      // 0〜100：達成度
  achieved: boolean;     // 十分な情報が得られたか
}
```

## 5.3 生成ルール

| ルール | 説明 |
| --- | --- |
| 生成タイミング | セッション開始時に 1 つ生成（`AIStateManager.create()`） |
| セッション中は固定 | `category` は会話中に変更しない（Phase3 で複数 Goal 対応予定） |
| 進捗更新 | 毎ターン `StateCalculator` が `progress` を更新 |
| 非公開 | UI・利用者への返答に Goal の存在を明かさない |

## 5.4 カテゴリと説明文

| Category | 説明例 |
| --- | --- |
| `JOB` | 相手の仕事や働き方について自然に知りたい |
| `HOLIDAY` | 相手の休日の過ごし方について自然に知りたい |
| `MARRIAGE` | 相手の結婚観について自然に知りたい |
| `FAMILY` | 相手の家族との関係や家族観について自然に知りたい |
| `HOBBY` | 相手の趣味や好きなことについて自然に知りたい |
| `VALUE` | 相手の大切にしている価値観について自然に知りたい |
| `FOOD` | 相手の食の好みや食事の楽しみ方について自然に知りたい |
| `TRAVEL` | 相手の旅行の好みや行きたい場所について自然に知りたい |
| `FUTURE` | 相手の将来の展望や生活イメージについて自然に知りたい |
| `LOVE` | 相手の恋愛観について自然に知りたい |

## 5.5 DB マッピング

`ConversationSession.hiddenGoal`（JSON）と一致。

```json
{
  "category": "HOLIDAY",
  "description": "相手の休日の過ごし方について自然に知りたい",
  "progress": 60,
  "achieved": false
}
```

# 6. ConversationState 設計

## 6.1 定義

**ConversationState** は、会話セッションの進行状況を管理する。

## 6.2 構造

```typescript
interface ConversationState {
  currentTurn: number;    // 現在のターン数（AI 返答完了ごとに +1）
  minTurn: number;        // 最低ターン数（ホーム画面設定）
  maxTurn: number;        // 最大ターン数（ホーム画面設定）
  shouldEnd: boolean;     // 会話終了フラグ
  endReason: string;      // 終了理由（空文字 = 未終了）
}
```

## 6.3 初期値

| フィールド | 初期値 | 取得元 |
| --- | --- | --- |
| `currentTurn` | 0 | 固定 |
| `minTurn` | 10 | `HomeForm.conversationSetting.minTurn` |
| `maxTurn` | 20 | `HomeForm.conversationSetting.maxTurn` |
| `shouldEnd` | false | 固定 |
| `endReason` | `""` | 固定 |

## 6.4 終了判定

`05_AI設計.md` の会話終了判定に準拠する。

| 条件 | 説明 |
| --- | --- |
| 最大ターン到達 | `currentTurn >= maxTurn` → 必ず終了 |
| 最低ターン未満 | `currentTurn < minTurn` → 終了しない |
| 退屈・疲労 | `emotion.bored` が高い、または Relationship 低下 |
| Goal 達成 | `hiddenGoal.achieved === true` かつ自然な区切り |
| 話題の一区切り | 将来：`StateCalculator` が判定 |

## 6.5 DB マッピング

`ConversationSession` の `currentTurn` / `minTurn` / `maxTurn` / `status` と対応。

# 7. Memory 設計

## 7.1 定義

**Memory** は、会話を通じて蓄積される記憶である。

`05_AI設計.md` の Memory Manager 設計に準拠する。

## 7.2 構造

```typescript
interface FactMemory {
  id: string;
  content: string;
  createdAt: DateTime;
}

interface InsightMemory {
  id: string;
  content: string;
  createdAt: DateTime;
}

interface MemoryState {
  facts: FactMemory[];
  insights: InsightMemory[];
}
```

## 7.3 Fact と Insight の区別

| 種別 | 定義 | 例 |
| --- | --- | --- |
| `facts` | 利用者から得た**客観的事実** | 「犬を飼っている」「大阪在住」 |
| `insights` | 会話から**推測した内容** | 「家族思い」「インドア派」 |

## 7.4 更新ルール

| ルール | 説明 |
| --- | --- |
| 追加のみ（MVP） | 既存 Memory の削除・上書きは Phase2 以降 |
| 重複排除 | 同一内容の Fact は追加しない |
| 更新タイミング | 毎ターン終了後（`StateCalculator` または専用 MemoryUpdater） |
| Prompt 反映 | 最新 N 件を PromptBuilder が system prompt へ埋め込む |

## 7.5 初期値

```typescript
{ facts: [], insights: [] }
```

## 7.6 DB マッピング

`ConversationSession.memory`（JSON）と一致。

```json
{
  "facts": [
    { "id": "uuid", "content": "IT企業勤務", "createdAt": "..." }
  ],
  "insights": [
    { "id": "uuid", "content": "真面目な性格", "createdAt": "..." }
  ]
}
```

# 8. AIStateManager の責務

## 8.1 概要

`AIStateManager` は **AIState のライフサイクル管理** を担当する。

OpenAI や PromptBuilder は状態を**参照するのみ**とし、更新は Manager 経由で行う。

## 8.2 責務一覧

| 責務 | 説明 |
| --- | --- |
| 状態の保持 | `sessionId` をキーに `AIState` を保持する |
| 状態の生成 | セッション開始時に初期 `AIState` を生成する |
| 状態の取得 | 現在の `AIState` を返す |
| 状態の更新 | `StateCalculator` の結果をマージして保存する |
| 状態のリセット | セッション終了時に状態を破棄する |

## 8.3 公開メソッド

```typescript
class AIStateManager {
  create(sessionId: string, homeForm: HomeForm): AIState;
  get(sessionId: string): AIState | undefined;
  update(sessionId: string, context: UpdateContext): AIState;
  reset(sessionId: string): void;
}
```

## 8.4 UpdateContext

`update()` に渡すコンテキスト。`StateCalculator` の入力となる。

```typescript
interface UpdateContext {
  userMessage: string;
  assistantMessage: string;
}
```

## 8.5 処理フロー

```text
create()
  → 初期 Relationship / Emotion / HiddenGoal / Conversation / Memory を生成
  → Map に保存

get()
  → 現在の AIState を返す（PromptBuilder・EvaluationAI が参照）

update()
  → StateCalculator.calculate() を呼び出す
  → 差分をマージ
  → currentTurn++
  → 終了判定
  → Map を更新

reset()
  → Map から削除
```

## 8.6 保持場所

| フェーズ | 保持場所 |
| --- | --- |
| MVP（現状） | サーバーインメモリ（`Map<sessionId, AIState>`） |
| Phase2 | Prisma `ConversationSession` の JSON カラムへ永続化 |
| Phase3 | Repository 層経由で DB アクセス |

## 8.7 やってはいけないこと

| 禁止事項 | 理由 |
| --- | --- |
| OpenAI から状態を取得する | 状態の正はアプリ側 |
| PromptBuilder 内で状態を更新する | 責務分離違反 |
| Repository から直接 OpenAI を呼ぶ | 依存方向の逆転 |

# 9. StateCalculator の責務

## 9.1 概要

`StateCalculator` は、会話内容に基づき **AIState の差分（Delta）を計算する** 純粋な計算モジュールである。

状態を保持しない。副作用を持たない。

## 9.2 責務一覧

| 責務 | 説明 |
| --- | --- |
| Relationship 更新量の計算 | 利用者発言の質に応じた増減 |
| Emotion 更新量の計算 | 発言内容への感情反応 |
| HiddenGoal 進捗の計算 | 話題が Goal に関連するか判定 |
| Memory 追加の判定 | 新しい Fact / Insight の抽出 |
| 終了判定の補助 | `shouldEnd` / `endReason` の提案 |

## 9.3 公開メソッド

```typescript
class StateCalculator {
  calculate(
    currentState: AIState,
    context: UpdateContext,
  ): AIStateDelta;
}

interface AIStateDelta {
  relationship?: Partial<RelationshipState>;
  emotion?: Partial<EmotionState>;
  hiddenGoal?: Partial<HiddenGoalState>;
  conversation?: Partial<ConversationState>;
  memory?: {
    addFacts?: string[];
    addInsights?: string[];
  };
}
```

## 9.4 実装フェーズ

| フェーズ | 実装内容 |
| --- | --- |
| Phase2-2（現状） | 簡易ルール：`currentTurn++`、`nervous` 減少、`comfort` 上昇 |
| Phase2-3 | キーワードベースの Relationship / Emotion 更新 |
| Phase3 | LLM による構造化出力で Memory / Goal 進捗を判定 |
| Phase4 | 機械学習・ルールエンジンによる高度な判定 |

## 9.5 計算例（将来）

| 利用者発言 | 影響 |
| --- | --- |
| 趣味について具体的に話す | `relationship.interest +3`, `emotion.curious +5`, `hiddenGoal.progress +10` |
| 一方的な話 | `emotion.bored +5`, `relationship.interest -2` |
| 共感的な返答 | `relationship.trust +3`, `emotion.happy +5` |
| 失礼な発言 | `relationship.respect -5`, `emotion.nervous +3` |

# 10. PromptBuilder との連携

## 10.1 基本方針

`PromptBuilder` は `AIState` を**受け取り**、Markdown テンプレートへ値を埋め込む。

状態の更新は行わない。

## 10.2 入力

```typescript
interface BuildConversationMessagesInput {
  session: Session;           // プロフィール・性格・難易度
  aiState: AIState;           // 心理状態全体
  conversationHistory: ConversationHistoryMessage[];
  latestMessage: string;
}
```

## 10.3 テンプレート埋め込み項目

`src/prompts/conversation/system.md` および分割テンプレート（`06_プロンプト設計.md` 参照）へ以下を埋め込む。

| プレースホルダー | 埋め込み元 |
| --- | --- |
| `{{female_profile}}` | `Session.homeForm.femaleProfile` |
| `{{user_profile}}` | `Session.homeForm.userProfile` |
| `{{personality}}` | `Session.homeForm.personalitySetting` |
| `{{conversation_style}}` | 同上 |
| `{{difficulty}}` | 同上 + 行動レベル定義 |
| `{{hidden_goal}}` | `aiState.hiddenGoal.description` |
| `{{interest}}` | `aiState.relationship.interest` |
| `{{trust}}` | `aiState.relationship.trust` |
| `{{comfort}}` | `aiState.relationship.comfort` |
| `{{respect}}` | `aiState.relationship.respect` |
| `{{attraction}}` | `aiState.relationship.attraction` |
| `{{happy}}` | `aiState.emotion.happy` |
| `{{nervous}}` | `aiState.emotion.nervous` |
| `{{curious}}` | `aiState.emotion.curious` |
| `{{bored}}` | `aiState.emotion.bored` |
| `{{surprised}}` | `aiState.emotion.surprised` |
| `{{memory_facts}}` | `aiState.memory.facts` を整形 |
| `{{memory_insights}}` | `aiState.memory.insights` を整形 |

## 10.4 重要ルール

| ルール | 説明 |
| --- | --- |
| 数値は非公開指示 | 「数値をユーザーに言うな」とテンプレートに明記 |
| 固定値禁止 | すべて `AIState` から動的に取得 |
| Markdown 管理 | プロンプト文字列を TypeScript に直接書かない |
| messages 配列 | `[system, ...history, user(latest)]` の順で構築 |

## 10.5 処理フロー

```text
ConversationAI.reply()
  ↓
AIStateManager.get(sessionId)
  ↓
PromptBuilder.buildConversationMessages({ session, aiState, history, latestMessage })
  ↓
OpenAIClient.chat(messages)
  ↓
AIStateManager.update(sessionId, { userMessage, assistantMessage })
```

# 11. 将来 EvaluationAI との連携

## 11.1 基本方針

会話終了後、`EvaluationAI` は **最終的な `AIState` と会話履歴** を入力として評価を生成する。

EvaluationAI は状態を更新しない。読み取り専用で参照する。

## 11.2 入力

```typescript
interface EvaluationAIInput {
  session: Session;
  aiState: AIState;              // 最終状態
  conversationHistory: Message[];
}
```

## 11.3 EvaluationAI が参照する AIState 項目

| 参照項目 | 評価への活用 |
| --- | --- |
| `relationship` 最終値 | 「また会いたい度」「恋愛対象度」の根拠 |
| `emotion` 推移（履歴があれば） | 会話の楽しさ・緊張の変化の分析 |
| `hiddenGoal.progress` / `achieved` | 「休日の話題に触れられなかった」等の女性目線コメント |
| `memory.facts` / `insights` | 利用者の自己開示・共感の評価 |
| `conversation.endReason` | 会話終了理由の表示 |

## 11.4 処理フロー

```text
会話終了（conversation.shouldEnd === true）
  ↓
AIStateManager.get(sessionId)    … 最終状態を取得
  ↓
EvaluationAI.evaluate({ session, aiState, history })
  ↓
EvaluationResult 生成
  ↓
AIStateManager.reset(sessionId)  … 状態破棄（任意）
```

## 11.5 DB 永続化

| データ | 保存先 |
| --- | --- |
| 最終 `AIState` | `ConversationSession` の各 JSON カラム |
| 評価結果 | `Evaluation.result`（JSON） |

## 11.6 設計上の注意

| 注意 | 説明 |
| --- | --- |
| EvaluationAI は ConversationAI の役割を侵さない | 評価のみ。会話の続きを生成しない |
| 状態の正は AIStateManager | EvaluationAI が独自に状態を再計算しない |
| Hidden Goal は評価理由に使う | 利用者への会話中には明かさない |

# 13. 設計思想

AIState は「AI チャットの設定」ではなく、**一人の婚活女性の心の状態**を表現する。

OpenAI はその状態を読み取り、一貫した人物として返答を生成する。

状態の生成・更新・保持はすべてアプリケーション側の責務とし、
PromptBuilder はその状態を言語化して OpenAI に渡す橋渡しに徹する。

この分離により、将来の Emotion 解析・Memory 拡張・Evaluation 連携を、
OpenAI プロンプトの変更なしに段階的に実装できる。
