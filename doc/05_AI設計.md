# 05_AI設計.md

# 婚活AIトレーナー

Version: 2.0

---

# 1. AI設計方針

本システムでは、AIを単なるチャットボットではなく、一人の婚活女性として振る舞うことを目的とする。

AIは会話を行うだけでなく、

* 人物設定
* 感情
* 印象
* 記憶
* 会話の目的

を保持し、それらを基に自然なコミュニケーションを行う。

AIの各機能は責務ごとに分離し、保守性・拡張性・AI品質を高める。

---

# 3. AI一覧

| AI              | 役割                             |
| --------------- | ------------------------------ |
| Profile AI      | AI女性プロフィール・人格・Hidden Goalを生成する |
| Conversation AI | 婚活女性として会話を行う                   |
| Memory Manager  | 記憶・感情・印象を管理する                  |
| Evaluation AI   | 婚活アドバイザーとして会話を評価する             |

---

# 4. Profile AI

## 目的

AI女性の初期設定を生成する。

---

## 生成対象

### 基本プロフィール

* 年齢
* 仕事
* 居住地
* 趣味
* その他情報

---

### 人格

* 性格
* 会話スタイル
* 価値観
* 難易度

---

### Hidden Goal

利用者には公開しない会話目的を生成する。

例

* 家族を大切にする人か知りたい
* 趣味が合う人か知りたい
* 仕事と家庭の考え方を知りたい
* 誠実な人か確認したい

Conversation AIは、このHidden Goalを意識しながら会話を進める。

---

# 5. Conversation AI

## 役割

婚活女性として自然な会話を行う。

---

## 入力

* UserProfile
* FemaleProfile
* PersonalityProfile
* HiddenGoal
* Memory
* EmotionStatus
* ImpressionStatus
* ConversationHistory
* UserMessage

---

## 出力

* AI返信
* 会話終了判定

---

## 会話ルール

Conversation AIは以下を遵守する。

* 女性として振る舞う
* 一貫した人格を維持する
* Hidden Goalを自然に確認する
* 過去の会話内容を考慮する
* 知らない情報は推測しない
* 評価は行わない

---

# 6. Memory Manager

## 目的

Conversation AIが利用する情報を整理・更新する。

---

## 管理対象

### Fact

利用者から得た客観的な事実。

例

* 犬を飼っている
* 奈良県在住
* IT企業勤務

---

### Insight

AIが会話から推測した内容。

例

* 家族思い
* 真面目な性格
* アウトドア好き

---

### EmotionStatus

会話中に変化する短期的な感情。

```text
Interest（興味）
Comfort（安心感）
Enjoyment（楽しさ）
Fatigue（疲労）
```

特徴

* 毎ターン更新する
* 次の返答内容へ強く影響する

---

### ImpressionStatus

会話全体を通して形成される印象。

```text
Trust（信頼感）
Sincerity（誠実さ）
Compatibility（価値観の一致）
Attractiveness（魅力）
Romance（恋愛対象度）
```

特徴

* 急激には変化しない
* 会話全体を通じて少しずつ更新される
* 評価画面の主要な指標となる

---

# 7. Memory更新ルール

各ターン終了後にMemory Managerが実行される。

更新対象

* Fact追加・更新
* Insight追加・更新
* EmotionStatus更新
* ImpressionStatus更新

Conversation AIは更新済みの状態のみ参照する。

---

# 8. Hidden Goal

## 概要

AI女性は会話開始時に一つ以上のHidden Goalを持つ。

利用者には公開されない。

Conversation AIは、この目的を達成するために自然な質問や話題を選択する。

---

## 例

* 家族観を知りたい
* 金銭感覚を知りたい
* 趣味が合うか知りたい
* 子どもについて考えを知りたい
* 休日の過ごし方を知りたい

---

## 評価への反映

評価画面では、

* Hidden Goalを達成できたか
* 十分な情報を得られたか

も評価理由として利用する。

例

「私は休日の過ごし方を重視していましたが、その話題に触れられなかったため、相性を判断しづらいと感じました。」

---

# 9. 会話終了判定

Conversation AIは以下を総合的に判断する。

* 最低ターンを満たしている
* 最大ターンへ到達した
* Fatigueが高い
* 話題が一区切りついた
* Hidden Goalの確認が完了した

自然な流れで会話を終了する。

---

# 10. Evaluation AI

## 目的

婚活アドバイザーとして利用者へ**厳しく現実的な**フィードバックを行う。

「優しいAI」ではなく、**「もう一度会いたい」と思わせられる会話か**を基準に評価する。

---

## 採点方式（Version 2.5）

**減点方式**を採用する。各項目は満点からスタートし、問題に応じて減点する。**加点方式は禁止**。

| 項目 | 満点 |
| --- | --- |
| 共感力 | 20 |
| 質問力 | 20 |
| 自己開示 | 15 |
| 深掘り力 | 15 |
| 会話の自然さ | 15 |
| 婚活らしさ | 15 |
| **合計** | **100** |

### 難易度補正（サーバー側）

| 難易度 | 補正 |
| --- | --- |
| Easy | +15点 |
| Normal | 補正なし |
| Hard | -10点 |
| Extreme（将来） | -20点 |

Normal では普通の会話が **45〜60点** 程度、**70点超えは稀**とする。

---

## 判定基準

| 点数 | 判定 |
| --- | --- |
| 90〜100 | ぜひまた会いたい（★★★★★） |
| 75〜89 | もう一度会ってみたい（★★★★☆） |
| 60〜74 | 迷う（★★★☆☆） |
| 40〜59 | 厳しい（★★☆☆☆） |
| 0〜39 | お断りしたい（★☆☆☆☆） |

---

## 出力（Evaluation）

* 項目別スコア（`itemScores`）
* 総合点（難易度補正後 `score`）
* 総評（改善点重視）
* 女性心理（文章）
* 改善ポイント + 模範回答
* 次回の課題
* もう一度会いたい確率（0〜100%）
* 判定（星評価）

---

## 追加分析

* AIState（Interest / Trust / Comfort / Romance）の参考
* Hidden Goal の達成状況
* 会話全体の流れ

---

## 責務分離

Evaluation AI

* 評価のみ担当
* 会話の続きを生成しない

---

## Character Aware Evaluation（Version 2.6）

Session から**性格・会話スタイル・Hidden Goal・難易度**を取得し、相手に合わせた会話力を評価する。

| 要素 | 内容 |
| --- | --- |
| 性格別採点 | `evaluationCharacterRules.ts` で期待値・減点例を定義 |
| 会話スタイル補正 | よく話す / 聞き上手 / バランス型 |
| Hidden Goal 補正 | Goal に触れられない場合は減点 |
| characterAdaptationScore | 性格適合度 0〜100（LLM 評価） |

詳細は `doc/21_EvaluationAI_性格考慮型評価_V2.6.md` を参照。

各AIは他AIの責務へ介入しない。

---

# 12. AI品質方針

AIは利用者に「正解」を提示することを目的としない。

一人の女性として一貫した価値観・感情・印象・目的を持ち、その人物らしい判断を行う。

利用者は様々な価値観を持つ相手との会話を経験し、自身のコミュニケーション能力を向上させることを目指す。
