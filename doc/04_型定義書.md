# 1. 設計方針

本プロジェクトでは、TypeScriptによる型安全を重視する。

以下のルールに従って型を設計する。

* Domain ModelとDTOを分離する
* any型は禁止
* Enumを積極的に利用する
* Optionalは必要最小限とする
* APIレスポンスはすべて型定義を持つ

# 17. 設計思想

型は「現在必要なもの」だけではなく、「将来の拡張」を考慮して設計する。

すべてのAI処理・API通信・画面表示は、この型定義を基準とし、システム全体で一貫したデータ構造を維持する。
