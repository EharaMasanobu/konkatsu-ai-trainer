# 1. システム構成

本システムはクライアント・サーバー構成を採用する。

```text
┌─────────────────────┐
│      React UI       │
└─────────┬───────────┘
          │ HTTP
┌─────────▼───────────┐
│ Express API Server  │
├─────────────────────┤
│ ConversationService │
│ PromptService       │
│ EvaluationService   │
│ ProfileService      │
│ SpeechService       │
└─────────┬───────────┘
          │
 ┌────────┴──────────┐
 │                   │
▼                    ▼
OpenAI API        SQLite
```

# 8. エラーハンドリング

対象

* API通信失敗
* JSONパース失敗
* 音声認識失敗
* OpenAIエラー

利用者へは分かりやすいメッセージを表示する。

---

# 9. ログ設計

開発時は以下を出力する。

* API送信時間
* API応答時間
* Promptサイズ
* Token数
* エラー内容

本番では個人情報を含むログは保存しない。

---

# 10. セキュリティ

* APIキーはサーバー側のみ保持
* クライアントへ秘密情報を渡さない
* HTTPS前提
* 入力値はすべてバリデーションする

---

# 11. 拡張性

以下の追加を容易にする設計とする。

* 新しいシナリオ
* AI人格追加
* 評価ロジック追加
* 音声合成
* 多言語対応

---

# 12. パフォーマンス方針

* Promptを必要最小限に構成する
* Memoryを活用しToken消費を抑える
* 非同期通信を採用する
* UIの再描画を最小限にする

---

# 13. 設計原則

* 単一責任の原則（SRP）
* 型安全を維持する
* Promptとロジックを分離する
* AI依存処理をサービス層へ集約する
* 将来的なシナリオ追加を考慮した構成とする
