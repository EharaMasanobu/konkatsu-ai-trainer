# 1. 設計方針

MVPでは SQLite + Prisma を採用する。

会話中に頻繁に更新されるデータはJSONとして保持し、
将来的な拡張時に正規化できる構成とする。

# 2. 使用技術

- SQLite
- Prisma ORM

開発環境ではSQLiteを利用し、
将来的にPostgreSQLへ移行できる設計とする。

# 6. Prismaモデル

```prisma
model ConversationSession {

  id String @id @default(uuid())

  createdAt DateTime @default(now())

  updatedAt DateTime @updatedAt

  status String

  currentTurn Int

  minTurn Int

  maxTurn Int

  userProfile Json

  femaleProfile Json

  personality Json

  hiddenGoal Json

  conversationHistory Json

  memory Json

  emotion Json

  impression Json

  evaluation Evaluation?
}

model Evaluation {

  id String @id @default(uuid())

  createdAt DateTime @default(now())

  sessionId String @unique

  result Json

  session ConversationSession @relation(fields:[sessionId], references:[id])
}
```
